# RL_single_cell

This repository contains code and analyses developed across multiple research threads, centered around (i) drug–gene target aggregation and validation, and (ii) computational search strategies for **cell-state conversion** in single-cell perturbation embedding spaces.

---

## Repository overview

### `Isoforms paper analysis/`
Analyses in this folder relate to the following manuscript:

**Jon Bezney\***, **Ziming Weng\***, Carlo Ruggeri, Maggie Maurer, Stanley Qi, Lars Steinmetz, Matthew Wheeler, Stephen Montgomery.  
*“Isoform aware CRISPR depletion of whole-blood RNA-seq for enhanced transcriptional phenotyping of rare Mendelian disorders.”*

(\* equal contribution)

> Note: This repository includes analysis code and notebooks used during the project. Data availability and final results should be interpreted in the context of the manuscript and its associated data-sharing plan.

---

### `drugs_to_genes/`
Code to **merge multiple drug–gene target resources** into a unified drug→target-gene mapping dataset.

- Includes preprocessing, harmonization, and merging logic across datasets.
- See the README inside `drugs_to_genes/` for details on sources, rules used for merging, and outputs.

---

### Search-based cell-state conversion
We implemented several strategies to identify **sequences of perturbations** that move a cell from a starting state to a desired target state (in an embedding space such as PCA):

- **Greedy search** (e.g., beam/tree variants)
- **Probabilistic search** (sampling-based)
- **Reinforcement learning** (DQN-style agent)

Relevant code and results are primarily in:

- `analysis/` — notebooks, evaluation, figures, scaling analyses, and RL experiments
- `searches/` — core search algorithm implementations and utilities

---

### `data_and_models/`
This folder contains datasets and intermediate artifacts used by the project, including:

- processed / subsetted Tahoe data used in downstream experiments
- outputs generated by the search procedures 
- other serialized objects required for analyses

> Because some upstream datasets are very large (e.g., Tahoe-100M), not all raw inputs may be tracked directly in Git. Where relevant, scripts assume local paths or precomputed subsets.
